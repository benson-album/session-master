## 0. QA Agent 工作指引

### QA Agent 的核心职责

QA Agent（功能测试代理）是本项目的**质量守门员**，每个里程碑签字前必须由 QA Agent 完成验收。

| 职责 | 执行方式 |
|:-----|:---------|
| **验证开发计划中的 T?-V 步骤** | 每个子任务完成后执行其验证步骤（T1-V、T2-V 等） |
| **执行本测试文档中的用例** | 按 14 模块分组逐条执行 |
| **回归测试** | 批次间确保新代码不破坏已有功能 |
| **缺陷记录** | 发现失败用例时记录：复现步骤 / 预期 / 实际 / 影响范围 |
| **里程碑签字** | 批次内所有用例通过后在开发计划对应的 check box 打 ✅ |

### QA Agent 工作流程

```
接收到 PM Agent 的测试任务
  │
  ▼
读取当前版本代码（`src/` 目录）
  │
  ▼
执行对应模块的所有手工/自动测试用例
  │
  ▼
┌─── 全部通过？ ───┐
│ 是               │  否
│                  │
▼                  ▼
写入测试报告       记录缺陷 → 通知 DE Agent 修复
(VERIFIED)        → DE 修复后重跑
  │
  ▼
PM Agent 收到 VERIFIED 状态 → 进入下一个任务

> **注意**：40+ 测试用例目前为手动测试，首次完整执行估算约 2h。
>
> **Service Worker 生命周期提醒**：测试时注意 SW 休眠后恢复 alarm/P2P 状态的场景。
