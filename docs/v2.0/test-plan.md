# SessionMaster v2.0 — 测试文档

> **版本**：v2.0-draft
> **关联 PRD**：`docs/v2.0/PRD.md`
> **测试类型**：全面验收测试
> **测试环境**：Chrome / Edge Manifest V3

---

## 1. 测试范围

### 1.1 覆盖范围

| 模块 | 测试重点 |
|------|---------|
| 🌐 站点选择器 | 下拉切换、添加站点、跨Tab同步 |
| 🌐 会话管理 Tab | Cookie/保活/拦截按站点隔离显示 |
| 🔄 同步管理 Tab | 主从+P2P/服务器按站点独立配置 |
| ⚙️ 全局设置 Tab | 配置读取/保存/跨站点共享 |
| 🔒 锁定规则 | 无站点→锁定、有站点→解锁 |
| 🔄 数据迁移 | v1.x → v2.0 存储自动迁移 |
| 🔄 回滚方案 | 旧键保留，迁移可逆 |

### 1.2 不覆盖范围

- 后台功能逻辑（Cookie CRUD、WebRTC、加密等）— 与 v1.x 相同

---

## 2. 测试场景定义

| 场景 | 前置条件 | 预期 |
|:----:|---------|------|
| **A — 全新安装** | 无任何存储数据 | 站点列表为空，引导添加 |
| **B — 从 v1.x 升级** | 存在 `heartbeat_configs` + `sync_config` | 数据自动迁移到 `v2_sites` + `v2_global` |
| **C — 空白页** | `about:blank` 或 `chrome://newtab` | 站点选择器可用，全局设置可用 |
| **D — 正常页（已有站点）** | `https://oa.company.com`，且该站点已在 v2 存储中 | 全部功能可用 |
| **E — 正常页（新站点）** | `https://new-site.com`，不在 v2 存储中 | 自动识别，提示添加 |
| **F — 多站点** | 已配置 3 个以上站点 | 下拉切换流畅，数据独立 |

## 4. 边界条件

| ID | 测试项 | 前提 | 预期 |
|:--:|--------|------|------|
| B-1 | 空域名站点 | 用户输入空字符串添加站点 | 校验拒绝 |
| B-2 | 重复站点 | 添加已存在的域名 | 提示已存在 |
| B-3 | 站点数上限 | 添加超过 50 个站点 | 性能无明显下降 |
| B-4 | 保活 URL 为空 | 添加保活时未输入 URL | toast 提示 |
| B-5 | 无保活记录 | 站点下有 0 条保活 | 显示占位提示 |
| B-6 | 迁移时 heartbeat_configs 为空 | 无保活但有同步配置 | 迁移后站点只有同步配置 |
| B-7 | 迁移时 sync 为空 | 有保活但无同步配置 | 迁移后站点只有保活，同步默认关闭 |
| B-8 | 浏览器拓展重载 | 升级后 SW 重启 | 配置不丢失，正常加载 |

## 5. 验收标准

| # | 条件 | 验证方式 |
|:-:|------|---------|
| 1 | 全部 TC-SEL ~ TC-MIG 测试通过 | 逐条执行，无 FAIL |
| 2 | v1.x → v2.0 迁移后功能可用 | 从正式环境导出 → 安装 v2.0 → 验证全部功能 |
| 3 | 控制台无 TypeError | 各场景轮换至少 30 秒 |
| 4 | 构建脚本通过 | `bash scripts/build.sh` 返回 0 |
| 5 | 9 项强制自检通过 | 见标准开发流程第 6 阶段 |
| 6 | 空白页下半数功能可用 | 站点选择 + 全局设置正常工作 |
