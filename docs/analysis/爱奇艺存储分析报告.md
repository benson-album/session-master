### 3.2 关键发现 1 — 专用指纹 SDK（Gaze v1.0.58）

爱奇艺使用独立部署的指纹 SDK（`stc.iqiyipic.com/gaze/fingerprint/`），是本次分析的所有站点中**指纹采集最完备**的系统：

| 采集函数 | 采集内容 | 隐身模式敏感度 |
|:---------|:---------|:--------------|
| `getCanvasFingerprint()` | Canvas 2D 渲染哈希 | ⚠️ 高（隐身模式可能不同） |
| `getAudioFingerprint()` | OfflineAudioContext 波形特征 | ⚠️ 高（隐身模式可能不同） |
| `getWebglFingerprint()` | WebGL 渲染器信息 | 🟡 中（驱动相关） |
| `getFonts()` | 系统字体列表 | ✅ 稳定 |
| `getFontPreferences()` | 字体渲染偏好 | ✅ 稳定 |
| `getScreenResolution()` | 屏幕分辨率 | ✅ 稳定 |
| `getScreenFrame()` | 屏幕可用区域 | ✅ 稳定 |
| `getRoundedScreenFrame()` | 取整后的屏幕尺寸 | ✅ 更稳定（防指纹绕过） |
| `getDeviceMemory()` | 设备内存（navigator.deviceMemory） | ✅ 稳定 |
| `getMathFingerprint()` | Math 函数精度差异（acos/acosh/sin 等） | ✅ 极稳定 |
| `getPcaDeviceId()` | PCA 设备 ID（从 QC005_NATIVE Cookie） | ✅ 持久化 Cookie |
| `getFingerprint()` | **主入口**，聚合所有指纹生成唯一标识 | — |

**指纹 SDK 特点：**
- 使用 `OfflineAudioContext` 采集音频指纹（比腾讯视频的 `getAudioContext` 更先进）
- Canvas 指纹是异步的（`__awaiter` 模式）
- 有 Math 函数指纹——通过分析不同浏览器对浮点数计算的精度差异来识别浏览器
- 指纹聚合后存储在 `__dfp`（localStorage）中
- 另有 `adapter_dfp` 记录指纹适配器信息（`fp1` + `fp2`）

> ⚠️ **对插件影响**：爱奇艺的指纹系统在所有分析的站点中最完整，达到业界顶级水平（类似 FingerprintJS Pro）。隐身/普通模式切换时，Canvas + Audio 指纹可能变化，加上 PCA 设备 ID 交叉验证，**跨模式迁移失败风险极高**。

### 3.3 关键发现 2 — QC 系列 Cookie 体系

爱奇艺使用 **QC 系列** Cookie 作为设备/会话标识体系：

| Cookie | 用途 | 来源 |
|:-------|:------|:------|
| `QC005` | 设备 ID（核心指纹关联） | qysdk 内 `GlobalData.deviceId` |
| `QC006` | 会话 ID | 安全 SDK 设置 |
| `QC007` | 会话辅助标识 | 安全 SDK 设置 |
| `QC008` | 扩展标识 | 安全 SDK 设置 |
| `QC234` | 设备标识（与 localStorage 的 QC234 同值） | qysdk |
| `QC235` | 设备标识（与 localStorage 的 QC235 同值） | qysdk |
| `__dfp` | 设备指纹哈希 | 指纹 SDK 写入 |
| `__uuid` | 浏览器 UUID | 服务端或本地生成 |

> `QC005` 与 `getPcaDeviceId()` 函数关联，是 PCA（平台认证）设备标识的核心。存储在 Cookie 中以便跨请求传递。

### 3.4 关键发现 3 — qysdk 登录/认证体系

qysdk（爱奇艺 SDK）中包含完整的登录认证逻辑：

```javascript
// 认证 Cookie 读取
auth: null !== c.P00001 ? c.P00001 : null

// 设备 ID 与 QC005 关联
QC005: l.GlobalData.deviceId

// 登录响应处理（P00036 Cookie）
function procLoginResp(e) {
    var t = decodeURIComponent(getCookie("P00036"));
    // 从 Cookie 中提取 token
    var token = p.toObj(t).token || e.data.token || "";
}
```

| Cookie | HttpOnly | 用途 |
|:-------|:--------:|:------|
| `P00001` | ✅ | 会话令牌（核心认证） |
| `P00002` | ✅ | 用户令牌 |
| `P00003` | ❌ | 前一用户标识（`pu`） |
| `P00036` | ❌ | 登录响应数据（含 token） |
| `P00037` | ❌ | 登录响应码 |
| `P00038` | ❌ | 登录响应消息 |

> **双写检测**：未发现 qysdk 有明显的 localStorage ↔ Cookie 双写认证机制。认证主要基于 Cookie（P00001~P00003 系列），localStorage 中的 `risc_loginInfo` 存储的是风险控制数据而非认证令牌。

### 3.5 关键发现 4 — Aegis + Beacon 监控标识

爱奇艺同时使用了 **Tencent Aegis** 和 **自研 Beacon** 两套监控系统：

| 系统 | localStorage Key | 值格式 | 用途 |
|:-----|:-----------------|:-------|:------|
| **Aegis** | `AEGIS_ID` | UUID v4 | 腾讯监控设备 ID |
| **Beacon** | `__BEACON_deviceId` | 短哈希字符串 | 爱奇艺自研埋点设备 ID |
| **Beacon** | `__BEACON_0WEB0C0CG154MK0F_*` | 多种会话指标 | 埋点会话数据 |

## 五、设备指纹汇总

| 模块 | 采集函数 | 隐身模式敏感度 | 说明 |
|:-----|:---------|:--------------|:------|
| **Canvas** | `getCanvasFingerprint()` | 🔴 高 | Canvas 2D 渲染哈希，隐身模式可能变化 |
| **Audio** | `getAudioFingerprint()` | 🔴 高 | OfflineAudioContext，最敏感的指纹之一 |
| **WebGL** | qysdk 内部 | 🟡 中 | WebGL 渲染器信息 |
| **字体** | `getFonts()` | ✅ 稳定 | 系统字体列表 |
| **Math** | `getMathFingerprint()` | ✅ 极稳定 | 浮点精度差异 |
| **PCA 设备 ID** | `getPcaDeviceId()` → QC005 Cookie | ✅ 持久化 | 服务端分配的持久 ID |
| **Aegis ID** | localStorage AEGIS_ID | 🟡 中 | UUID，隐身模式隔离 |

> 爱奇艺的指纹体系是本次分析中**最完善**的（比腾讯视频的 vendor-common.js 采集维度更多），涵盖了腾讯视频有的 Canvas/WebGL/Audio + 独有的 Math 指纹 + PCA 设备 ID。

## 六、跨模式迁移风险评估

| 因素 | 风险 | 说明 |
|:-----|:----|:------|
| Cookie 认证 | ⚠️ 中 | P00001 HttpOnly Cookie 可导出，但指纹 SDK 交叉验证 |
| 设备指纹 | 🔴 **高风险** | Canvas + Audio 双指纹在隐身模式下均可能变化 |
| PCA 设备 ID | ✅ 低风险 | QC005 Cookie 持久化，导出后可恢复 |
| Math 指纹 | ✅ 极稳定 | 跨模式不变 |
| Aegis + Beacon | ⚠️ 中 | localStorage 隐身模式下隔离 |

## 八、结论

爱奇艺在本次分析的所有站点中拥有**最完善的设备指纹系统**（Gaze v1.0.58），涵盖 Canvas、Audio、WebGL、Math、字体等 10+ 维度。认证基于 P00001~P00003 系列 Cookie，localStorage 主要存储指纹数据和风控信息。

关键发现：

| 维度 | 爱奇艺 | 与其他站点对比 |
|:-----|:-------|:-------------|
| 指纹 SDK | **Gaze v1.0.58** 独立部署 | 优于腾讯视频的 vendor-common.js |
| 采集维度 | Canvas + Audio + WebGL + Math + 字体 | **最全面** |
| 双写机制 | ❌ 无 | 同百度网盘 |
| Cookie 体系 | QC 系列 + P 系列 + `__dfp` | 双体系，复杂度最高 |
| SSO | passport.iqiyi.com (Skywalker) | 独立于腾讯/百度/阿里体系 |
| 跨模式迁移 | 🔴 **最高风险** | Canvas + Audio 双指纹 + PCA 双重验证 |
