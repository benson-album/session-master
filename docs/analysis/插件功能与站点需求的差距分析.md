# 插件功能与站点需求的差距分析

> 日期：2026-06-22

## 一、预设字段不足

当前 `storage_presets.json` 仅有 4 个字段，无法充分描述站点特性：

| 当前字段 | 欠缺字段 | 影响 |
|:---------|:---------|:------|
| — | `ssoGroup: string` (SSO体系归类: 百度/阿里/腾讯/自有) | 无法自动关联跨域SSO兄弟站 |
| — | `authType: "cookie"\|"localStorage"\|"hybrid"\|"unknown"` | 导入策略只能一刀切 |
| — | `hasDeviceFingerprint: bool` | Cookie值含指纹签名时导入会失效 |
| — | `knownCookiePrefixes` | 仅前缀匹配，缺少精确Cookie名列表 |

## 二、认证体系类型分类

| 类型 | 站点举例 | 难点 | 插件兼容性 |
|:-----|:---------|:-----|:-----------|
| 纯 Cookie | 哔哩哔哩、微博 | HttpOnly Cookie可通过 API 导出 | ✅ 完整兼容 |
| localStorage Token | 飞书、钉钉 | 需登录后采集Key名 | ⚠️ 部分兼容（需预设） |
| JSON嵌套 | 腾讯视频(ams_cookies) | 单Key含多认证凭证 | ⚠️ 整体复制不做深解析 |
| 混合型(Cookie+localStorage) | 腾讯视频、百度系 | 需要两边同时导出 | ✅ Phase 3已兼容 |
| 指纹绑定 | 夸克(ctoken)、常见于中国站 | Cookie值含设备指纹 | ❌ 导入后可能不工作 |

## 三、SSO跨域扫描

站点有SSO关联的列表：

| 站点 | SSO体系 | 关联域 | 现状 |
|:-----|:--------|:-------|:-----|
| 优酷(youku.com) | 阿里系 | .taobao.com | ✅ cookieDomains已列 |
| 百度网盘(pan.baidu.com) | 百度系 | .baidu.com | ✅ 已列 |
| 腾讯微云(weiyun.com) | 腾讯系 | .qq.com | ✅ 已列 |
| 爱奇艺(iqiyi.com) | 自有+百度联合 | .baidu.com (备) | ❌ cookieDomains未列.baidu.com |

**修复建议：爱奇艺的 cookieDomains 应追加 `.baidu.com`**

## 四、高置信度 vs 低置信度

登录态能确认的（置信高）：
- 哔哩哔哩: ✅
- 百度: ✅
- 微博: ✅
- 腾讯视频: ✅ (ams_cookies已确认)
- 飞书/钉钉/Notion: ⚠️ (常识推测)

需要登录确认的（置信低，标注为猜想）：
- 阿里云盘: localStorageKeys为推测
- 夸克网盘: localStorageKeys为推测
- 腾讯微云: localStorageKeys空
- 天翼云盘: 全部空
- 迅雷云盘: 全部空

## 五、结论与改进建议

| 优先级 | 改进项 | 难度 |
|:-------|:-------|:-----|
| P0 | 增加 `ssoGroup` + `authType` 字段到预设结构 | 💧 低 |
| P1 | 完善爱奇艺的 cookieDomains（追加.baidu.com） | 💧 低 |
| P1 | 增加设备指纹标记字段 | 💧 低 |
| P2 | JSON嵌套Key深解析（ams_cookies拆分为子条目） | 🔥 高 |
| P2 | 登录态确认完成后补充低置信度预设 | 💧 低 |
| P3 | SSO跨域自动探测（访问A站时顺带扫SSO域） | 🔥 高 |
