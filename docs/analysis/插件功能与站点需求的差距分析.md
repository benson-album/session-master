# 插件功能与站点需求的差距分析

> **日期**：2026-06-22（初版）/ 2026-06-24（深度分析升级后更新）

## 一、预设字段不足

当前 `storage_presets.json` 仅有 4 个字段，无法充分描述站点特性：

| 当前字段 | 欠缺字段 | 影响 |
|:---------|:---------|:------|
| — | `ssoGroup: string` (SSO体系归类: 百度/阿里/腾讯/自有) | 无法自动关联跨域SSO兄弟站 |
| — | `authType: "cookie"\|"localStorage"\|"hybrid"\|"unknown"` | 导入策略只能一刀切 |
| — | `hasDeviceFingerprint: bool` | Cookie值含指纹签名时导入会失效 |
| — | `fingerprintLevel: "none"\|"light"\|"medium"\|"extreme"` | 无法评估跨模式迁移风险 |
| — | `knownCookiePrefixes` | 仅前缀匹配，缺少精确Cookie名列表 |

## 七、结论与改进建议

### 深度分析核心发现

1. **指纹系统差异极大**：爱奇艺（extreme）→ B站（none），插件需按级别采取不同策略
2. **初版报告 Key 误判较多**：`browserId`、`bdstoken`、`loginData` 等实际非 localStorage Key
3. **SSO 独立性**：爱奇艺并非百度 SSO 关联（初版误判），使用自研 Skywalker
4. **双写机制非普遍现象**：仅腾讯视频确认有 localStorage ↔ Cookie 双写，其他站点均无

### 优先级更新

| 优先级 | 改进项 | 难度 |
|:-------|:-------|:-----|
| **P0** | 增加 `ssoGroup` + `authType` + `fingerprintLevel` 字段到预设结构 | 💧 低 |
| **P0** | 将修正后的 Key 清单同步到 `storage_presets.json` | 💧 低 |
| **P1** | 你登录阿里云盘/夸克/天翼/迅雷/123 → 补全深度分析 | 💧 低（需要你操作） |
| **P1** | 增加设备指纹标记字段 | 💧 低 |
| **P2** | JSON嵌套Key深解析（ams_cookies拆分为子条目） | 🔥 高 |
| **P3** | SSO跨域自动探测（访问A站时顺带扫SSO域） | 🔥 高 |
