### 3.2 关键发现 1 — WebGL 设备指纹

百度网盘使用 **WebGL Canvas 指纹** 进行设备 GPU 类型检测，而非完整的 Canvas 哈希指纹（与腾讯视频/AWSC Fireye 不同）：

```javascript
function x() {
    try {
        var e = document.createElement("canvas").getContext("webgl");
        if (e) {
            var r = e.getExtension("WEBGL_debug_renderer_info");
            if (r) {
                var i = e.getParameter(r.UNMASKED_RENDERER_WEBGL) || "";
                // 检测 Apple Silicon / Intel / AMD GPU
                if (i.includes("Apple M")) return "apple";
                if (i.includes("Apple GPU") && !/safari/i.test(navigator.userAgent))
                    return "apple";
                if (i.includes("Intel") || i.includes("AMD")) return "intel";
            }
            // 备选：检测 WebGL 扩展组合
            var o = e.getSupportedExtensions().includes("WEBGL_compressed_texture_s3tc_srgb");
            var a = e.getSupportedExtensions().includes("WEBGL_metal");
            if (!o && a) return "apple";
        }
    } catch(e) {}
    // fallback: 从 UserAgent 判断
    var s = navigator.userAgent;
    return s.includes("ARM") || s.includes("aarch64") ? "arm" : "other";
}
```

**含义**：百度网盘使用 WebGL Canvas 做 GPU 类型识别（Apple/Intel/ARM），而非完整的 Canvas 指纹哈希。这种指纹在不同浏览器模式（隐身/普通）下 **相对稳定**，但仍可能因 GPU 驱动行为差异而变化。

### 3.3 关键发现 2 — `browserId` API 参数

**重要修正**：初版报告认为 `browserId` 是 localStorage Key，**实际运行时确认不在 localStorage 中**。`browserId` 是作为 **API 查询参数** 和 **请求体字段** 动态使用的：

```javascript
// browserId 作为 API 查询参数
fetch("/api/invoker/online?browserId=" + t)

// browserId 在请求体中传递
{ browserId: r, method: t }
{ browserId: i, seq: o }

// 初始化时的 browserId 状态
this.intermediateData = {
    browserId: "",   // 初始为空，运行时动态生成
    seq: "",
    version: "",
    online: 2,
    port: 0
}
```

`browserId` 主要用于 **桌面客户端互通**（百度云管家）：

| API | 用途 | 参数 |
|:----|:------|:-----|
| `/api/invoker/online` | 检测桌面客户端是否在线 | `browserId` |
| `/api/invoker/send` | 推送下载任务到客户端 | `browserId`, `downloadInfo` |
| `/api/invoker/get` | 获取下载进度 | `browserId`, `seq` |
| `/api/invoker/check` | 检查下载状态 | `browserId`, `seq` |

> ⚠️ 对插件影响：`browserId` 不需纳入 localStorage 预设，它是运行时生成的临时设备标识，用于客户端通信。

### 3.4 关键发现 3 — 无 localStorage-Cookie 双写

百度网盘的 vendors 包中 **未发现** localStorage ↔ Cookie 的双写机制。认证体系：

- **Cookie 驱动**：BAIDUID 是全局认证种子
- **bdstoken**：从 `/api/loginStatus` 接口返回，不是 localStorage Key
- **PANPSC**：网盘专属 Cookie（HttpOnly）
- **csrfToken**：CSRF 保护 Cookie

与腾讯视频不同，百度网盘的认证 **完全基于 Cookie**，localStorage 不存储核心认证数据。

## 五、认证流程（完整版）

```
访问 pan.baidu.com
  │
  ├─→ 营销落地页加载
  │    ├─→ chunk-vendors (1.7MB) 加载
  │    ├─→ Sentry 初始化 → 生成 nd_sentry_did
  │    ├─→ 检查 BAIDUID / BAIDUID_BFESS Cookie
  │    └─→ 设置 csrfToken Cookie
  │
  ├─→ 检查 BAIDUID 是否存在
  │    │
  │    ├─→ 存在（已登录过）
  │    │    ├─→ 跳转 /disk/home（SPA 应用）
  │    │    ├─→ 加载 SPA JS bundles（需登录态才下发）
  │    │    ├─→ 调用 /api/loginStatus
  │    │    │    ├─→ 返回 bdstoken, uk, username, avatar
  │    │    │    └─→ 设置 PANPSC HttpOnly Cookie
  │    │    └─→ 渲染文件列表
  │    │
  │    └─→ 不存在（未登录）
  │         ├─→ 302 → /login?redirecturl=...
  │         ├─→ 选择 QR/账号/短信登录
  │         ├─→ passport.baidu.com 认证
  │         └─→ 回调设置 BAIDUID, BAIDUID_BFESS
  │
  └─→ 关键 Cookie 写入顺序
       ├─→ csrfToken（pan.baidu.com）— 登录页写入
       ├─→ BAIDUID（.baidu.com）— 百度 SSO 写入（登录后）
       ├─→ BAIDUID_BFESS（.baidu.com）— BAIDUID 备用
       └─→ PANPSC（pan.baidu.com, HttpOnly）— 网盘认证（登录后）
```

## 六、设备指纹汇总

| 模块 | 类型 | 隐身模式敏感度 | 说明 |
|:-----|:-----|:--------------|:------|
| **WebGL Canvas** | GPU 类型检测 | 🟢 低 | 检测 Apple/Intel/ARM GPU，相对稳定 |
| **nd_sentry_did** | Sentry 设备 UUID | 🟡 中 | Sentry 错误监控分配，隐身模式下可能重置 |
| **BAIDUID** | 持久化 Cookie | 🟢 稳定 | 百度 SSO 持久化，与模式无关 |

> 百度网盘的设备指纹体系相对较轻量，**隐身/普通模式切换风险低于腾讯视频和优酷**。WebGL 指纹仅用于 GPU 分类，不参与核心会话绑定。

## 七、跨模式迁移风险评估

| 因素 | 风险评估 | 说明 |
|:-----|:---------|:------|
| Cookie 认证 | ✅ 低风险 | 认证完全基于 Cookie，无双写依赖 |
| localStorage | ✅ 低风险 | 不存储核心认证数据 |
| 设备指纹 | 🟡 中风险 | WebGL Canvas + Sentry DID 在隐身模式下可能变化 |
| HttpOnly Cookie | ⚠️ 需注意 | BAIDUID, PANPSC 均为 HttpOnly，需 Chrome API 导出 |
| bdstoken | ✅ 低风险 | 由 API 返回，导入 Cookie 后自动获取 |

## 九、结论

百度网盘使用百度统一 SSO 认证体系（`passport.baidu.com`），认证完全基于 Cookie（`BAIDUID` + `PANPSC`），localStorage **不存储核心认证数据**。

主要发现更新：

| 维度 | 初版评估 | 深度分析后修正 |
|:-----|:---------|:--------------|
| `browserId` | localStorage Key ❌ | **API 查询参数** ✅ |
| `bdstoken` | localStorage Key ❌ | **API 返回值** ✅ |
| `loginData` / `baidu_token` | 列入预设 🟡 | **运行时不存在** — 已移除 |
| 登录态 localStorage | 9 个 Key | **仅 2 个**（passLoginType + nd_sentry_did） |
| 设备指纹 | 未分析 | **WebGL Canvas GPU 检测** |
| 双写机制 | 未分析 | **不存在** |
| 认证流程 | 线性 4 步 | **完整 12 步链路** |
