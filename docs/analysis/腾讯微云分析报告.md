# 腾讯微云 Cookie/存储分析报告

> 日期：2026-06-22  
> 工具：curl / whatweb / Kali Linux

## 一、HTTP 架构

| 端点 | 状态 | 服务器 | 说明 |
|:-----|:----:|:-------|:-----|
| `weiyun.com` | 302 | TSW | 跳转到 www.weiyun.com |
| `www.weiyun.com` | 200 | TAPISIX | 最终页面 |

> 重定向链路：weiyun.com → (302) → www.weiyun.com

## 二、安全头部

| 头部 | 值 |
|:-----|:----|
| `X-Frame-Options` | 缺失 ❌ |
| `X-XSS-Protection` | `1; mode=block` ✅ |
| `Strict-Transport-Security` | 缺失 ❌ |
| `X-Content-Type-Options` | `nosniff` ✅ |
| `Content-Security-Policy` | 缺失 ❌ |

> ⚠️ 虽有 X-XSS-Protection 和 X-Content-Type-Options，但缺少关键的 X-Frame-Options。

## 三、认证机制

### 3.1 Cookie 体系

| Cookie | 域 | HttpOnly | 用途 |
|:-------|:---|:--------:|:-----|
| `TOK` | `.weiyun.com` | ❌ | 会话认证 Cookie（需确认 HttpOnly） |
| `fingerprint` | `.weiyun.com` | ❌ | 设备指纹 Cookie |
| `tgw_l7_route` | `.weiyun.com` | ❌ | 七层反向代理路由标识 |

> `fingerprint` 为设备指纹 Cookie，用于浏览器标识追踪。

### 3.2 localStorage（预计，需登录确认）

| Key | 用途（推测） |
|:----|:-------------|
| `weiyun_token` | 微云认证令牌 |
| `weiyun_user` | 用户信息缓存 |

> 腾讯系产品预计沿用 `weiyun_` 前缀的 localStorage Key 命名风格。

### 3.3 技术栈

| 组件 | 说明 |
|:-----|:------|
| TSW | 腾讯自研 Web 服务器（Tencent Secure Web Server） |
| TAPISIX | Apache APISIX API 网关（腾讯定制版） |
| stgw | 腾讯自研七层反向代理（`tgw_l7_route` 来源） |
| SSO | 腾讯统一登录（login.qq.com） |

### 3.4 认证流程（推测）

```
访问 www.weiyun.com
  ↓
检查 TOK Cookie
  ↓
存在 → 携带 TOK 调用 TAPISIX API → 返回登录态
不存在 → login.qq.com → QQ扫码/账号登录
       ↓
回调设置 TOK、fingerprint 等 Cookie
```

## 四、关联 Cookie 域

- `.weiyun.com` — 微云主域
- `.qq.com` — 腾讯SSO跨域

## 六、结论

腾讯微云使用腾讯 SSO 体系（login.qq.com）。技术栈包含 TSW（腾讯自研 Web 服务器）+ TAPISIX API 网关 + stgw 反向代理。`TOK` 为会话认证 Cookie，`fingerprint` 为设备指纹。安全配置方面缺失 X-Frame-Options，存在一定安全风险。`tgw_l7_route` 为七层代理路由标识。需登录态确认 localStorage Key。
