### 3.2 关键发现 1 — Wbi 签名密钥系统

B站使用 Wbi 签名算法保护 API 请求。密钥通过图片 URL 传递：

```javascript
// Wbi 密钥管理（localStorage）
window.localStorage.setItem("wbi_img_url", L.wbi_img.img_url);
window.localStorage.setItem("wbi_sub_url", L.wbi_img.sub_url);

// localStorage 中的实际密钥格式
wbi_img_urls: "https://i0.hdslb.com/bfs/wbi/7cd084941338484aae1ad9425b84077c.png
               -https://i0.hdslb.com/bfs/wbi/4932caff0ff746eab6f01bf08b70ac45.png"

// 代码中提取后的密钥
wbiImgKey: "c6618eced9b64a019d0e6247aed7a38e"  // 从图片文件名提取
wbiSubKey: "6bb3adc89c244ccda6d9abf22fc6f910"  // 从图片文件名提取
```

| 属性 | 说明 |
|:-----|:------|
| **密钥源** | 图片 URL 文件名（定期轮换） |
| **存储方式** | localStorage `wbi_img_urls`（`-` 分隔） |
| **有效期** | 动态轮换，API `/x/web-interface/wbi/index/...` 获取最新 |
| **影响** | 签名密钥在 localStorage 中，跨模式导出可恢复 |

### 3.3 关键发现 2 — Canvas WebP 检测（非指纹）

B站的 `canvas` 使用是 **WebP 格式支持检测**，而非 Canvas 指纹采集：

```javascript
canvas.toDataURL("image/webp").indexOf("data:image/webp") === 0
```

此函数检查浏览器是否支持 WebP 图片格式，属于**功能检测**，不参与设备标识或认证绑定。

> ⚠️ 与其它站点的对比：B站没有 Canvas 哈希 / WebGL / Audio 指纹采集系统，仅有设备标识 Cookie（buvid 系列）。

### 3.4 关键发现 3 — 票据系统（bili_ticket）

B站使用票据系统进行安全风控：

| Cookie | 说明 |
|:-------|:------|
| `bili_ticket` | 安全票据（有效期内） |
| `bili_ticket_expires` | 票据过期时间 |

票据由 `forticket.min.js` 或服务端下发，用于验证请求合法性。

## 五、设备指纹汇总

| 模块 | 类型 | 隐身模式敏感度 | 说明 |
|:-----|:-----|:--------------|:------|
| **buvid3/4** | 持久化 Cookie | ✅ 稳定 | 服务端分配的设备标识 |
| **buvid_fp** | 浏览器指纹 Cookie | ✅ 稳定 | 浏览器级别指纹 |
| **`_uuid`** | UUID Cookie | 🟡 可能重置 | 隐身模式下可能新生成 |
| **bili_ticket** | 安全票据 | 🟡 需刷新 | 有时效性 |
| **Canvas** | 仅 WebP 检测 | ✅ 稳定 | **非指纹采集** |

> B站没有像腾讯视频/爱奇艺那样的完整 Canvas 哈希或 Audio 指纹采集。设备标识主要依赖 `buvid` 系列持久 Cookie。

## 六、跨模式迁移风险评估

| 因素 | 风险 | 说明 |
|:-----|:----|:------|
| Cookie 认证 | ✅ 低风险 | SESSDATA + bili_jct HttpOnly Cookie，导出后可直接恢复 |
| 设备指纹 | ✅ 低风险 | buvid 系列 Cookie 可导出恢复 |
| Canvas 指纹 | ✅ 不存在 | 仅有 WebP 格式检测 |
| Wbi 签名密钥 | ✅ 低风险 | localStorage 密钥可导出 |
| bili_ticket | 🟡 需注意 | 有时效性，导入后需刷新 |

## 八、结论

B站在本次分析的站点中**设备指纹最轻量**，没有 Canvas/WebGL/Audio 前端指纹采集。认证完全基于 HttpOnly Cookie（`SESSDATA` + `bili_jct`），localStorage 不存储核心认证数据。

| 维度 | B站 | 与其他站点对比 |
|:-----|:-----|:-------------|
| 设备指纹 | **未采集**（仅 WebP 检测） | 最轻量 |
| Wbi 签名 | 自研图片密钥系统 | ✅ 独特 |
| 双写机制 | ❌ 无 | 同百度网盘 |
| 安全票据 | bili_ticket | 有额外安全层 |
| 跨模式迁移 | ✅ **风险最低** | 优于腾讯视频/爱奇艺 |

## 退出机制

哔哩哔哩为 SPA 应用，退出通过 SDK 函数调用：

| 属性 | 值 |
|:-----|:----|
| **退出函数** | `Ke.logout(successCallback, currentUrl)` |
| **CSRF 保护** | 请求体携带 `biliCSRF: bili_jct` Cookie 值 |
| **对插件影响** | ⚠️ URL 无退出关键词，需要 SDK 函数拦截 |
| **插件方案** | 运行时重写 `Ke.logout` 方法，触发退出确认弹窗 |
