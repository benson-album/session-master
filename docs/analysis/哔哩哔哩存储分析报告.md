> 日期：2026-06-22  
> 工具：curl / whatweb / Kali Linux

## 一、HTTP 架构

| 端点 | 状态 | 服务器 | 说明 |
|:-----|:----:|:-------|:-----|
| `www.bilibili.com` | 200 | nginx/openresty | 主站 SPA |
| `passport.bilibili.com` | 302 → /site/site.html | openresty | SSO 登录 |
| `api.bilibili.com` | 200 | nginx | REST API |
| `s1.hdslb.com` | 200 | CDN | 静态资源 |

## 三、认证机制

### 3.1 Cookie 体系

| Cookie | 域 | HttpOnly | 用途 |
|:-------|:---|:--------:|:-----|
| `SESSDATA` | `.bilibili.com` | ✅ | 会话数据（核心认证） |
| `bili_jct` | `.bilibili.com` | ✅ | CSRF 令牌 |
| `DedeUserID` | `.bilibili.com` | ✅ | 用户ID |
| `DedeUserID__ckMd5` | `.bilibili.com` | ✅ | 用户ID校验和 |
| `buvid3` | `.bilibili.com` | ❌ | 设备唯一标识 v3 |
| `buvid4` | `.bilibili.com` | ❌ | 设备唯一标识 v4 |
| `b_lsid` | `.bilibili.com` | ❌ | 本地会话ID |
| `b_nut` | `.bilibili.com` | ❌ | 用户编号 |
| `buvid_fp` | `.bilibili.com` | ❌ | 浏览器指纹 |

### 3.2 localStorage

| Key | 用途 |
|:----|:------|
| `ac_time_value` | 反爬虫时间戳 |
| `buvid3` | 设备标识缓存 |
| `buvid4` | 设备标识缓存 |
| 其他配置项 | 播放设置、主题配置 |

### 3.3 API 签名

所有 API 请求需携带 `w_rid`（基于 Wbi 签名算法）+ `wts`（时间戳）。签名密钥通过 `api.bilibili.com/x/web-interface/nav` 获取，定期轮换。

### 3.4 认证流程

```
访问 bilibili.com
  ↓
检查 SESSDATA + bili_jct Cookie
  ↓
存在 → api.bilibili.com/x/web-interface/nav → 返回登录态
不存在 → 302 到 passport.bilibili.com → 登录 → 设置 Cookie
```

## 四、关联 Cookie 域

- `.bilibili.com` — 主域
- `.hdslb.com` — 静态资源 CDN

## 五、已知 localStorage Key

```
ac_time_value, buvid3, buvid4, b_lsid
```

## 六、结论

B站认证依赖 HttpOnly Cookie（`SESSDATA` + `bili_jct`），localStorage 不包含核心认证数据。插件需要导出的主要是 `.bilibili.com` 域的 Cookie。
