# 夸克网盘 Cookie/存储分析报告

> 日期：2026-06-22  
> 工具：curl / whatweb / Kali Linux

## 一、HTTP 架构

| 端点 | 状态 | 服务器 | 说明 |
|:-----|:----:|:-------|:-----|
| `pan.quark.cn` | 200 | Tengine (Alibaba-Aliyun) | 直接200，无重定向 |

## 二、安全头部

| 头部 | 值 |
|:-----|:----|
| `X-Frame-Options` | `SAMEORIGIN` ✅ |
| `X-XSS-Protection` | 缺失 ❌ |
| `Strict-Transport-Security` | 存在 ✅ |
| `X-Content-Type-Options` | 缺失 ❌ |
| `Content-Security-Policy` | 缺失 ❌ |

> 夸克网盘安全配置在网盘中属于较优水平。HSTS 和 X-Frame-Options 均已配置。

## 三、认证机制

### 3.1 Cookie 体系

| Cookie | 域 | HttpOnly | 用途 |
|:-------|:---|:--------:|:-----|
| `ctoken` | `.quark.cn` | ❌ | 会话安全令牌（每次请求都下发） |

> **说明：** `ctoken` 是会话标识而非认证 Cookie 本身。认证 Cookie 需登录态确认。

### 3.2 localStorage（预计，需登录确认）

| Key | 用途（推测） |
|:----|:-------------|
| `quark_token` | 夸克认证令牌 |
| `quark_user` | 用户信息缓存 |

> 夸克属于阿里系产品，预计沿用阿里体系 localStorage 命名风格。

### 3.3 频率限制

| 状态 | 服务器 | 说明 |
|:----:|:-------|:------|
| 503 | Tengine/AServer-Ingress | 触发限流后的返回 |

> 频繁请求会触发 Tengine/AServer-Ingress 的 503 限流。

### 3.4 技术栈

- Web 服务器：Tengine（阿里自研）
- 限流：AServer-Ingress
- SSO：阿里系，移动端跳转 `h5.sm.cn`
- 关联域：`.uc.cn`、`.sm.cn`

### 3.5 认证流程（推测）

```
访问 pan.quark.cn
  ↓
检查 ctoken Cookie 及 localStorage Token
  ↓
存在 → 携带 Token 调用 API → 返回登录态
不存在 → h5.sm.cn（移动端）→ 扫码/账号登录
       ↓
回调设置 ctoken + localStorage Key
```

## 四、关联 Cookie 域

- `.quark.cn` — 主域
- `.uc.cn` — 阿里系关联
- `.sm.cn` — 移动端登录跳转

## 六、结论

夸克网盘采用阿里系技术栈（Tengine），安全头部配置较优（HSTS ✅、X-Frame-Options: SAMEORIGIN ✅）。`ctoken` 为会话安全令牌，每次请求携带。存在 Tengine/AServer-Ingress 层限流（503）。认证体系与阿里系一致，移动端跳转 `h5.sm.cn`。localStorage 需登录确认。
