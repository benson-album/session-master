### 3.2 关键发现 1 — Mtop 双写机制（`syncCookieMode`）

`vendors.chunk.js` 中发现阿里系 **Cookie ↔ Cookie 同步** 机制，与腾讯视频的 `setStorageItem` 不同，优酷使用 Mtop 中间件的 `syncCookieMode`：

```javascript
// vendors.chunk.js 中发现的 Mtop 同步逻辑
f.config.mainDomain = "youku.com";
f.config.syncCookieMode = true;

// 认证失败时自动重试（含 Cookie 同步）
if (r.syncCookieMode && result.indexOf("ILLEGAL_ACCESS") > -1) {
    // 重新处理 Cookie 后重试请求
    r.__cookieProcessor = ...;
    r.failTimes = 0;
}
```

**含义**：优酷的 Cookie 管理由 Mtop 中间件统一处理，认证失败时会自动触发 `__cookieProcessor` 重试 Cookie 同步。这与腾讯视频的 `localStorage ↔ Cookie` 双写机制不同——优酷的认证信息**主要集中在 Cookie 中**，而非依赖 localStorage 备份认证头。

> ⚠️ **对插件的影响**：优酷的认证完全基于 Cookie（`_m_h5_tk` 等），localStorage 不存储核心认证数据。SessionMaster 导出 Cookie 时应能正常工作，但需注意 `_m_h5_tk` 等阿里系 Cookie 的 HttpOnly 属性。

### 3.3 关键发现 2 — AWSC 设备指纹系统

AWSC（Alibaba Web Security Component）含有完整的设备指纹采集系统：

| 模块 | 用途 | 说明 |
|:-----|:------|:-----|
| **UAB** | 用户行为分析 | 鼠标轨迹、点击行为 |
| **Fireye** | 风控指纹 | Canvas/WebGL/Audio 指纹 |
| **UMID Token** | 唯一设备标识 | `umidToken = "defaultToken1_um_not_loaded@@" + 随机值`;
AWSC 加载后通过回调更新 |
| **NS Module** | 网络安全 | 用于防篡改 |

```javascript
// AWSC 初始化代码
var i = {};
i.umidToken = "defaultToken1_um_not_loaded@@" + r + "@@" + (new Date).getTime();
e.AWSC.use("um", function(e, t) {
    if ("loaded" === e) {
        // UMID 加载成功后更新 token
    }
});
```

**含义**：优酷（阿里系）使用 AWSC 采集设备指纹，包含 Canvas/WebGL/Audio 等特征。这与腾讯视频的指纹系统属于平行设计，**隐身/普通模式切换时指纹可能变化**，从而被风控判定为新设备。

### 3.4 关键发现 3 — `deviceId` 生成函数

```javascript
function G() {
    var t = navigator.userAgent;
    if ((null == t ? void 0 : t.indexOf("YoukuDesktop")) > -1) {
        var e = t.indexOf("UTDID"), r = t.indexOf(";", e);
        return t.slice(e + 6, r).trim().replace(/(^\s*)|(\s*$)/g, "") || "";
    }
    // 返回空字符串或通过其他方式生成 deviceId
}

// 使用方式
deviceId: G() || "XlQcF5xQrCcCAWoLKdGqIOhS"
```

**含义**：优酷桌面客户端通过 UserAgent 中的 `UTDID` 提取设备 ID。Web 端回退到硬编码默认值 `"XlQcF5xQrCcCAWoLKdGqIOhS"`。这意味着**浏览器指纹的主要采集渠道是 AWSC/Fireye**，而非客户端层面的 UTDID。

## 七、退出机制分析

### 7.1 退出链路

优酷为阿里系 SPA，退出通过 SDK 发起自定义协议跳转：

```
用户点击「退出」
  |
  +---> YKLoginLoader SDK 调用
  |    +---> YKLoginLoader({loginOrLogout:"logout", buid, pid, logoutSuccess})
  |         +---> 成功回调中设置 window.location.href = "passport://logout"
  |              自定义协议，唤起优酷客户端
  |
  +---> fallback（无客户端时）
  |    +---> account.youku.com/cross.htm?forceLogin=true
  |         +---> 销毁服务端 Session
  |
  +---> Cookie 清除
       +---> 清除 _m_h5_tk、_tb_token_、cookie2
       +---> 跳转到登录页
```

### 7.2 SDK 函数签名

| 属性 | 值 |
|:-----|:----|
| **SDK** | `YKLoginLoader`（内联在 vendors.chunk.js） |
| **退出参数** | `{loginOrLogout:"logout", buid, pid, logoutSuccess: fn}` |
| **自定义协议** | `passport://logout` |
| **HTTP 兜底** | `account.youku.com/cross.htm?forceLogin=true&pid=20160607PLF000287` |
| **退出状态** | 内部 `logoutStatus` 对象跟踪各回调完成状态 |

### 7.3 对插件退出保护的影响

| 维度 | 评估 |
|:-----|:------|
| XHR URL 拦截 | `cross.htm?forceLogin=true` 不含退出关键词 |
| href 跳转拦截 | 已覆盖。`passport://logout` 通过 `location.href` setter 拦截 |

## 八、结论

优酷使用阿里系完整技术栈（Mtop + AWSC + Tengine）。与腾讯视频不同，优酷**没有** `localStorage ↔ Cookie` 的双写认证机制——认证完全依赖 HttpOnly Cookie（`_m_h5_tk`）。localStorage 中存储的是安全令牌（`tfstk__`）和播放器密钥（`YK_PLAYER_CKEY`），而非认证令牌备份。

主要发现对比：

| 维度 | 腾讯视频 | 优酷 |
|:-----|:---------|:-----|
| 双写机制 | `setStorageItem` localStorage ↔ Cookie | Mtop `syncCookieMode` Cookie 仅同步 |
| 核心认证 | `ams_cookies` localStorage + `uin` Cookie | `_m_h5_tk` HttpOnly Cookie |
| 设备指纹 | `vendor-common.js` 手写指纹采集 | AWSC Fireye 全套阿里风控体系 |
| SSO | ptlogin2.qq.com | cnpassport.youku.com (Havana) |
| API 协议 | TRPC (pbaccess.video.qq.com) | Mtop (阿里移动网关) |
| 跨模式迁移风险 | ✅ 高（双写 + 指纹） | ⚠️ 中（指纹，无双写） |

> ⚠️ **跨模式迁移评估**：优酷的 Cookie + AWSC 指纹体系在隐身/普通模式切换时，`__ysuid`、`__aysid` 等会话 Cookie 会重置，但核心 `_m_h5_tk` Cookie 如果正确导出了 HttpOnly 版本，理论上应能工作。主要风险来自 AWSC Fireye 指纹差异可能导致风控拦截（阿里系对指纹变化较为敏感）。

-----|:----|
| **退出方式** | `window.location.href = "passport://logout"`（自定义协议） |
| **底层 SDK** | `YKLoginLoader({loginOrLogout:"logout", ...})` |
| **对插件影响** | ⚠️ 非标准 HTTP URL，需要拦截 `window.location.href` 赋值 |
| **插件方案** | `Object.defineProperty` 重写 `location.href` setter，匹配 `passport://logout` 时弹出确认 |
