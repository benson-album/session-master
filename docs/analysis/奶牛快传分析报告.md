## 三、认证机制

### 3.1 Cookie 体系

| Cookie | 域 | HttpOnly | 用途 |
|:-------|:---|:--------:|:-----|
| 无 | — | — | 未登录态下无Cookie下发 |

> 奶牛快传非典型"网盘"，实际为文件传输工具，设计上不持久保存用户登录态。

### 3.2 localStorage（可能不涉及长登录态）

> 作为文件传输工具，可能仅在会话期间使用临时存储，不涉及持久登录态。

### 3.3 技术栈

| 组件 | 说明 |
|:-----|:------|
| 前端 | 纯静态页面 |
| 安全配置 | 无 |
| SSO | 无（非持久存储服务） |

> **提示：** 奶牛快传（cowtransfer.com）本质上是**文件传输服务**（临时分享），而非传统意义上的持久存储网盘。其设计目标是一次性/限时文件分享，不涉及长期用户登录态管理，因此 cookie 和 localStorage 的持久化需求较低。这与百度网盘、阿里云盘等需要持久登录态的产品有本质区别。

### 3.4 使用场景

```
用户 A → 上传文件 → 生成分享链接/提取码
                         ↓
用户 B → 访问链接 → 输入提取码 → 下载文件
```

- 无需双方注册登录即可完成传输
- 文件有有效期（通常 7 天）
- 不持久保存用户文件

## 四、关联 Cookie 域

- `.cowtransfer.com` — 主域

## 五、Preset 配置建议

> 奶牛快传不是典型的持久存储网盘，暂不加入预设列表。

## 六、结论

奶牛快传并非传统意义上的"网盘"，而是一个**文件传输服务**。其设计理念为临时文件分享，不持久保存用户登录态。使用纯静态页面，无安全头部配置，未登录态无 Cookie 下发。与百度网盘、阿里云盘等需要长期登录态管理的持久存储网盘有本质区别。因此不建议将其加入持久存储预设列表。
