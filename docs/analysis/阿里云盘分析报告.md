# 阿里云盘 Cookie/存储分析报告

> 日期：2026-06-22  
> 工具：curl / whatweb / Kali Linux

## 一、HTTP 架构

| 端点 | 状态 | 服务器 | 说明 |
|:-----|:----:|:-------|:-----|
| `aliyundrive.com` | 301 | istio-envoy / Tengine | 跳转到 www.aliyundrive.com |
| `www.aliyundrive.com` | 301 | istio-envoy / Tengine | 跳转到 www.alipan.com |
| `www.alipan.com` | 200 | istio-envoy / Tengine | 当前主域（最终页面） |

> ⚠️ **注意：** `aliyundrive.com` 已废弃，主域已迁移至 `www.alipan.com`。共经历三次 301 跳转。

## 三、认证机制

### 3.1 Cookie 体系

| Cookie | 域 | HttpOnly | 用途 |
|:-------|:---|:--------:|:-----|
| 无 | — | — | 未登录态下无Cookie下发 |

> 登录后预计会有阿里系 Cookie，如 `_m_h5_tk`、`cookie2`、`_tb_token_` 等（需登录确认）。

### 3.2 localStorage（预计，需登录确认）

| Key | 用途（推测） |
|:----|:-------------|
| `access_token` | OAuth 访问令牌 |
| `refresh_token` | OAuth 刷新令牌 |
| `token_type` | 令牌类型（Bearer） |
| `user_info` | 用户信息缓存 |
| `device_id` | 设备标识 |

> 阿里云盘采用标准 OAuth SPA 模式，认证令牌存储在 localStorage 中。

### 3.3 签名机制

- **Mtop 协议**：令牌 + 时间戳 + 签名
- **x-canary**：灰度发布 Header

### 3.4 技术栈

- 反向代理：istio-envoy service mesh
- Web 服务器：Tengine（阿里自研）
- API 网关：Mtop（阿里巴巴移动网关）
- SSO：阿里统一登录（taobao.com 体系）
- 架构：SPA（单页应用）

### 3.5 认证流程（推测）

```
访问 www.alipan.com
  ↓
检查 localStorage 中的 access_token
  ↓
存在 → 携带 Mtop 签名调用 API → 返回登录态
不存在 → login.taobao.com → 扫码/账号登录
       ↓
回调设置 localStorage Token（access_token, refresh_token）
```

## 四、关联 Cookie 域

- `.alipan.com` — 当前主域
- `.aliyundrive.com` — 旧域（已废弃）
- `.taobao.com` — 阿里SSO跨域

## 六、结论

阿里云盘已从 `aliyundrive.com` 迁移至 `www.alipan.com`，使用阿里统一登录体系（taobao.com）。采用 SPA 架构 + Mtop 协议签名。HSTS 配置良好（15552000秒），X-Frame-Options 为 `deny`。未登录状态下无 Cookie 下发。localStorage Token 模式需登录后确认具体 Key 名。
