# SessionMaster · 会话大师 更新日志

## v1.2.0 (2026-06-26)
### ✨ 新增功能
- **保活功能后端实现**：新增 `addHeartbeat()`、`removeHeartbeat()`、`toggleHeartbeat()`、`performHeartbeat()` 完整后端，通过 `chrome.alarms` 定时发送保活请求（10秒超时）
- **保活 UI 管理**：添加后可看到保活列表，支持✅启用/⏸️暂停/✕删除操作
- **保活状态栏**：显示共几条保活规则、几条活跃

### 🔧 优化改进
- **Tab 结构重塑**：从 4 个 Tab 精简为 3 个，Tab 1「🍪 会话」与 Tab 4「💓 保活」合并为「🍪 会话管理」
- **Tab 命名优化**：从「🍪 会话」改为「🍪 会话管理」，更精确反映包含 Cookie + 保活双重功能
- **帮助文档同步**：所有 Tab 引用更新为新结构，保活使用步骤更新
- **技能同步更新**：`sessionmaster-plugin` 技能更新为 3-Tab 架构描述

### 🐛 修复
- 保活按钮之前只弹出 toast，无实际效果——现已连接到真实后端
- 页面初始化时自动加载已保存的保活规则
- 插件安装时恢复保活定时器（`chrome.runtime.onInstalled`）

## v1.1.0 (2026-06-20)
### ✨ 新增功能
- **P2P 直连同步**：默认同步模式，浏览器间点对点传输 Cookie，无需中间服务器
- **信令服务器**：集成到同步服务器中，HTTP 长轮询实现，零额外依赖
- **WebRTC 引擎**：RTCPeerConnection + RTCDataChannel 实现浏览器直连
- **AES-256-GCM 加密**：配对码派生加密密钥，直连数据全程加密
- **网络地址检测**：双保险策略——`chrome.system.network` 优先 + WebRTC ICE 备用
- **IPv6 支持**：修复 IPv6 地址提取乱码问题，支持 12 种 ICE candidate 格式
- **端口自定义**：通过 `PORT` 环境变量自定义同步服务器端口（默认 5789）
- **帮助文档下载区**：支持一键下载 server.js、Dockerfile 等部署文件
- **版权与免责声明**：文件头部版权注释 + 帮助文档底部完整版权及免责说明
- **问号帮助图标**：信令服务器地址、服务器地址旁添加 ❔ 图标，点击跳转帮助文档

### 🔧 优化改进
- 插件界面全中文化（标签、标题、按钮、提示）
- 帮助文档脱敏处理（替换真实 OA 地址、账号密码、NAS IP 等）
- 移除文档中的敏感字（Kali → NAS、云服务器）
- 网络地址区段从 P2P 区域移出，放在同步 Tab 底部，双模式下均可见
- Hermes Gateway 系统提示消息改为中文（共 10 条）
- 文档保持专业风格，补充信令服务器技术原理说明

### 🐛 修复
- manifest.json 空字符串字段导致 Edge 加载失败
- Service Worker 中 WebRTC 受限导致地址获取失败（改用双保险策略）
- IPv6 地址误匹配 ICE candidate 中的数字字段（改为 token 分割校验）

## v1.0.0 (2026-06-20)
### ✨ 初始版本
- 基本框架搭建 — Manifest V3 浏览器扩展
- 🍪 **Cookie 导出/导入** — 手动同步功能
- 🛡️ **踢人请求拦截** — declarativeNetRequest + content script 双重拦截
- 💓 **会话保活** — 定期发送心跳防止 Cookie 过期
- ☁️ **云端自动同步** — AES-256-GCM 加密同步服务器
- **配对机制** — 8 位配对码（排除易混淆字符：`ABCDEFGHJKLMNPQRSTUVWXYZ23456789`）
- **自定义拦截规则** — 支持用户添加自定义 URL 关键词
- **帮助文档** — 完整的使用说明及部署指南
- **NAS Docker 部署** — 同步服务器部署在 Docker 容器中
- **安全设计** — Cookie 数据在浏览器本地加密，服务端只存密文
- 全部 62 项 Playwright 自测通过
