# SessionMaster · 会话大师 更新日志

## v1.3.0 (2026-06-21)
### 🔴 严重问题修复
- **Cookie 版本控制 + 来源追踪**：防多设备同步循环覆盖
  - 导出时标记每个 Cookie 的 `_exportTime` 时间戳
  - 导入时比较值+时间戳：值相同跳过，值不同且时间为旧则跳过
  - 远程导入的 Cookie 标记为 `origin=remote` → 自动同步时不再出口（防反弹）
  - OA 页面动态更新远程 Cookie 后自动转为 `local` 重新出口
  - 手动导入不受限制（用户主动信任）
- **Service Worker 重启后 P2P 连接状态重置**：启动时检测 `p2pConnected` 并重置，防止 UI 显示"已连接"但实际通道已断
- **所有 chrome.runtime.sendMessage 加 .catch()**：防止 popup 未打开时 SW 抛异常

### ✨ 新增功能
- **📡 主从设备模式（顶层控制开关）**
  - 开启后分为主设备（上传+下载）和从设备（仅接收不下发）
  - 关闭时自动版本控制（平等模式）
  - 可随时切换身份，同步Tab顶部显示当前状态
- **后台断开通知**：P2P 连接断开时通过 chrome.notifications 推送系统通知

### 🔧 优化改进
- **内容脚本拦截策略优化**：
  - 使用正则整词匹配，避免部分匹配误杀（如 `getSessionTimeout` 不再匹配）
  - 增加 `safeFnStr()` 安全获取函数体，防止 `toString` 抛异常
  - 拦截范围缩窄到 `800ms-3000ms`，减少对正常轮询的影响
  - 函数体超过 5000 字符不检查（压缩代码）
- **p2pPairKey 自动生成**：创建/加入房间时随机生成 16 位配对密钥
- **P2P 等待超时提示**：9分钟后显示红色警告

### 🗑️ 移除
- **injectSavedCookies**：功能不完整（只删 sessionStorage 未实际注入 Cookie），已移除

### 📋 新增权限
- `notifications`：后台连接断开时弹出系统通知

## v1.1.0 (2026-06-20)
### ✨ 新增功能
- **P2P 直连同步**：默认同步模式，浏览器间点对点传输 Cookie，无需中间服务器
- **信令服务器**：集成到同步服务器中，HTTP 长轮询实现，零额外依赖
- **WebRTC 引擎**：RTCPeerConnection + RTCDataChannel 实现浏览器直连
- **AES-256-GCM 加密**：配对码派生加密密钥，直连数据全程加密
- **网络地址检测**：双保险策略——`chrome.system.network` 优先 + WebRTC ICE 备用
- **IPv6 支持**：修复 IPv6 地址提取乱码问题，支持 12 种 ICE candidate 格式
- **端口自定义**：通过 `PORT` 环境变量自定义同步服务器端口（默认 5789）
- **帮助文档下载区**：支持一键下载 server.js、Dockerfile 等部署文件
- **版权与免责声明**：文件头部版权注释 + 帮助文档底部完整版权及免责说明
- **问号帮助图标**：信令服务器地址、服务器地址旁添加 ❔ 图标，点击跳转帮助文档

### 🔧 优化改进
- 插件界面全中文化（标签、标题、按钮、提示）
- 帮助文档脱敏处理（替换真实 OA 地址、账号密码、NAS IP 等）
- 移除文档中的敏感字（Kali → NAS、云服务器）
- 网络地址区段从 P2P 区域移出，放在同步 Tab 底部，双模式下均可见
- Hermes Gateway 系统提示消息改为中文（共 10 条）
- 文档保持专业风格，补充信令服务器技术原理说明

### 🐛 修复
- manifest.json 空字符串字段导致 Edge 加载失败
- Service Worker 中 WebRTC 受限导致地址获取失败（改用双保险策略）
- IPv6 地址误匹配 ICE candidate 中的数字字段（改为 token 分割校验）
